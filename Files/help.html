What is a KSV file?

If you are developing on the B4X platform, you have access to a local SQLite database solution that is easy to implement, without actually owning Database software. This is called the KeyValueStore.

The KeyValueStore is a class that you implement. You can find more information on it here:

https://www.b4x.com/android/forum/threads/keyvaluestore-class.36683/




Open/Import KSV file.

Click on the "Find File" button, or click File->Open. Navigate to the KVS file your application created, then click to Open it.

Note: If you have a large file, a file with a lot of encrypted fields, or a file containing a lot of images, it will take several seconds for the application to load the data. Be patient.




Backup KSV file.

I'm a big fan of not making more work for myself. That's why I always like to backup files before making changes.

Click on Edit->Backup KSV File after you have loaded the file in the application. You can backup as many times as you want. The application will serialize the backup files.




Encrypted Records

A KVS file can have all records encrypted or only some records encrypted. Enter the password for your encrypted files in the password box. If you enter this after having loaded the file, click the Reload button. Again, be patiend. Encrypted files load a bit slower if they are large.

If your KVS data has different passwords, you will need to enter the password and click the Reload button for each different password. The table will only show the data being decrypted with the current password. If it is not present, or not valid, the value will not show.




Recent Items List

The application keeps track of the files you open. Just click the drop down list and select a previous file to reload it.

To clear the recent items list, click Edit->Clear Recent Items




Hide Records

Right click on the table ROW to select a record and then on the context menu click Hide.

This will hide the record from the current table view but will NOT delete the record from the KVS file. You can restore the record by clicking the Reload button.




Copy Records

Right click on the table CELL to select content and then on the context menu click Copy. The content of that cell is then accessible in your clipboard.




Edit Records

Right click on the table ROW to select a record and then on the context menu click Edit Record.

The key and the value will appear in the editable fields at the bottom of the application. Make your changes and click the Update button.

NOTE: This makes a permanent change to the KVS File that is currently loaded. Remember what I said about backing things up first?




Delete Record

Right click on the table ROW to select a record and then on the context menu click Delete from KVS. You will be prompted to confirm your decision. This will permanently delete the record from the loaded KVS file.

NOTE: This makes a permanent change to the KVS File that is currently loaded. Remember what I said about backing things up first?




Reload Records

At any time you can Reload the current KVS File being viewed by clicking the Reload button.

During table view updates, the application attempts to refresh the data being displayed. However, if after a few seconds your update does not appear, then click the Reload button.




Clear Data

Click on Edit->Clear Data to empty the table view. This does NOT affect the live KVS File. The application will always clear the table when it is loading a new KVS File or reloading the current KVS File.




Export KVS Data to Plain Text, CSV, JSON, XML

Once you have your KSV File data loaded into the table, you can click Export->{option} to export the data to a properly formatted text file for the format chosen.




Data Validation for CSV, JSON, XML

When you export your data to CSV, JSON, or XML, it is recommended that you validate the output before shipping it off to another application or another user.

Click on Export->Validate and then choose the validator. You will then be taken a website where you can copy and paste your formatted records, or upload the formatted records, to validate their structure.




Supported Data Structures

The table view supports Text, Numbers, Lists, Maps and Images.

For Lists and Maps, be very careful about editing those records. The structure they are presented in is different.

List Example
[Red, Orange, Yellow, Green, Blue, Indigo, Violet, Mnemonic: Run, Oh You Great Big Irish Varlet]

Map Example
{Name=Jim Melanson, Email=jmelanson1965@gmail.com, City=Cobourg, Province=Ontario, Postal Code=K9A-4J8, Bio URL=https://about.me/jim.melanson, Writing website=http://www.jimmelanson.ca}

The data table will only display images if they are *.jpg, *.jpeg, *.gif, *.png, *.bmp. For images to be displayed, the Keyname MUST end with the appropriate extension. See the example files.

You cannot currently edit or add an image through the application. Yet.
